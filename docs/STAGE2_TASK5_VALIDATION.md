## 🔍 상세 검증 분석

### 1. 내 이벤트 페이지 (`/protected/events/page.tsx`)

**UX 흐름:**

1. 페이지 제목 (내 이벤트)
2. "내가 주최한 모임" 섹션
3. "참여 중인 모임" 섹션

**평가:**

- ✅ 탭 없음 (섹션 분리) - 명확함
- ✅ "새 모임 만들기" CTA 버튼 명확
- ✅ 빈 상태 메시지 친절함
- ⚠️ 섹션 간 시각적 구분 부족

**개선사항:** P2-001 (선택사항)

**통과율:** 95% 🟡

### 2. 모임 생성 페이지 (`/protected/events/new/page.tsx`)

**UX 흐름:**

1. 페이지 제목 + 설명 (하위 제목)
2. 폼 (제목, 카테고리, 날짜/시간, 장소, 최대 인원, 계좌 정보)
3. 제출 버튼

**문제점:**

- ❌ **폼 필드 순서 모호** (계좌 정보가 선택 항목이지만 맨 아래)
- ❌ **필수/선택 표시 부족** (\* 표시는 있지만 명확하지 않음)
- ⚠️ **폼 제출 후 피드백** (alert만 사용, 토스트 없음)

**개선사항:** P2-002 (필수)

- 필수 표시를 "필수" 텍스트로 명시 (현재: \*)
- 또는 "선택사항" 배지 추가

**통과율:** 90% 🟡

---

### 3. 모임 홈 페이지 (`/protected/events/[eventId]/page.tsx`)

**UX 흐름:**

1. 뒤로가기 버튼
2. 모임 제목 + 설명
3. 정보 카드 (날짜, 장소, 참여자, 초대 링크)
4. 핀 공지 (있으면)
5. 탭 네비게이션 (홈, 공지, 참여자, 카풀, 정산)
6. 탭 콘텐츠

**평가:**

- ✅ 정보 계층 명확
- ✅ 핀 공지 강조
- ⚠️ 탭 네비게이션 위치
  - 현재: 핀 공지 아래
  - 기대값: 정보 카드 위 또는 제목 아래

**개선사항:** P2-003 (선택사항)

- 탭 네비게이션을 정보 카드 위로 이동하여 페이지 섹션 명확화

**통과율:** 93% 🟡

### 4. 참여자 관리 페이지 (`/protected/events/[eventId]/members/page.tsx`)

**UX 흐름:**

1. 페이지 제목
2. 상태 필터 탭 (전체, 대기, 승인, 거절)

**문제점:**

- ⚠️ **빈 상태 메시지** (상태별 빈 상태 메시지 없음)

**개선사항:** P2-004 (필수)

- 상태별 빈 상태 메시지 추가
  - "대기 중인 신청이 없습니다"
  - "승인된 참여자가 없습니다"

**통과율:** 92% 🟡

## 🛠️ 필수 개선사항

### P2-002: 폼 필수/선택 표시 명확화

**파일:** `app/protected/events/new/page.tsx`
**변경:** FormLabel에서 "필수" 텍스트 추가
**예시:**

```tsx
// Before
<FormLabel>모임 제목 *</FormLabel>

// After
<FormLabel>모임 제목 <span className="text-red-600">*필수</span></FormLabel>
```

### P2-004: 상태별 빈 상태 메시지 추가

**파일:** `app/protected/events/[eventId]/members/page.tsx`
**변경:** 상태별 필터에 빈 상태 메시지 추가
**구현:**

```tsx
{
  memberList.length === 0 && (
    <div className="py-8 text-center">
      <p className="text-muted-foreground">
        {status === "pending" && "대기 중인 신청이 없습니다"}
        {status === "approved" && "승인된 참여자가 없습니다"}
        {status === "rejected" && "거절된 신청이 없습니다"}
      </p>
    </div>
  );
}
```
