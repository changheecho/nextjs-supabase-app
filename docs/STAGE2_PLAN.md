## 🎯 Task 분해 (8개)

### Task 1: 모바일 반응형 검증 (1.5일)

**목표**: 375px 뷰포트에서 터치 친화성, 텍스트 가독성, 레이아웃 검증

검증 대상 페이지:

- 내 이벤트 목록 (카드 너비, 탭 네비게이션, 버튼 크기)
- 모임 생성 폼 (입력 필드 높이, datetime-local 호환성)
- 모임 홈 페이지 (탭 스크롤, 정보 표시)
- 참여자 관리 (카드 레이아웃, 버튼 그룹)
- 공지 목록/작성 (카드 크기, textarea 높이)
- 초대 링크 페이지 (정보 미리보기, CTA 버튼)

**검증 도구**: Playwright MCP (375px 스크린샷)
**체크리스트**: 텍스트 줄바꿈, 버튼 크기(44x44px), 입력 필드(44px), 오류 메시지 가독성

---

### Task 2: 데스크톱/태블릿 레이아웃 검증 (1일)

**목표**: 768px+ 뷰포트에서 레이아웃 확장성, 여백, 다단 구조 검증

- 태블릿(768px): md:grid-cols-2, max-w-4xl, md:p-6
- 데스크톱(1024px): max-w-6xl, lg:p-8, 텍스트 라인 길이 최적화

**도구**: Playwright MCP (768px, 1024px, 1280px 스크린샷)

---

### Task 3: 색상 & 타이포그래피 일관성 검증 (0.5일)

**목표**: 전체 앱의 색상 사용, 글꼴 크기, 줄 높이 일관성 검증

검증 항목:

- 색상 역할: primary (생성 CTA), secondary (보조), destructive (거절)
- 배지 색상: 카테고리별 일관성 (모임/회의/행사/기타)
- 타이포그래피: h1(30px), h2(24px), text-lg(18px), text-base(16px), text-sm(14px)
- 다크모드: 색상 대비 WCAG AA 4.5:1 이상

**도구**: WebAIM Contrast Checker, 스크린샷 비교

---

### Task 4: 버튼 & 인터랙티브 요소 검증 (0.5일)

**목표**: 모든 버튼, 링크, 입력 필드의 접근성 및 사용성 검증

검증 항목:

- 버튼 크기: size="lg" (44px) for 모바일 CTA
- 버튼 상태: active (색상), hover (opacity), focus (ring-2), disabled (회색)
- 폼 입력: 라벨 명시, 필수 필드(\*) 마킹, 오류 메시지 위치, 포커스 피드백
- 링크: 색상 대비, underline/구분, 호버 피드백

---

### Task 5: 페이지별 UX 상세 검증 (1일)

**목표**: 각 페이지의 정보 구조, 흐름, 사용자 경험 상세 검증

7개 페이지 각각에 대해:

1. 정보 구조: 필요한 정보가 모두 보이는가?
2. 사용자 흐름: 자연스러운가? (주최자, 참여자 여정)
3. CTA 명확성: 버튼 텍스트가 명확한가?
4. 에러/빈 상태: 메시지가 명확하고 도움이 되는가?

---

### Task 6: 아이콘 & 시각 요소 일관성 (0.5일)

**목표**: lucide-react 아이콘 사용의 일관성 검증

- 크기 일관성: h-4 w-4 (기본), h-5 w-5 (강조)
- 색상 일관성: text-muted-foreground (보조), text-foreground (중요)
- 간격 일관성: gap-2 (8px)
- 사용 아이콘: Plus, MapPin, Users, ArrowRight, Copy, Check, X, Pin, Clock

---

### Task 7: 다크모드 호환성 검증 (0.5일)

**목표**: 라이트/다크 모드 모두에서 색상 대비 및 시각적 일관성 검증

- 배경색: bg-background, bg-card, bg-muted/50, bg-primary
- 텍스트: text-foreground, text-muted-foreground, text-primary-foreground
- 테마 전환 테스트: 라이트↔다크 모드 스크린샷 비교

---

### Task 8: 최종 개선사항 적용 & 확인 (1일)

**목표**: 검증 결과 문제점 수정 및 최종 품질 확보

작업:

1. 버그 수정: 반응형, 색상, 버튼, 텍스트 오버플로우
2. 개선사항 적용: 폼 에러, 카드 간격, 타이포그래피
3. 최종 검증: lint, tsc, 스크린샷 확인

## ✅ 완료 기준

### 기술 기준

- [ ] `npm run lint`: 오류 0건

- [ ] 모바일 375px: 정상 렌더링
- [ ] 태블릿 768px: 정상 렌더링
- [ ] 데스크톱 1024px: 정상 렌더링

### UX 기준

- [ ] 모든 CTA 버튼: 최소 44x44px 이상
- [ ] 모든 입력 필드: 최소 44px 높이
- [ ] 모든 텍스트: line-clamp로 오버플로우 처리
- [ ] 필수 필드: \* 마킹 명확
- [ ] 오류 메시지: 명확하고 가독성 있음

### 접근성 기준

- [ ] 색상 대비: WCAG AA 4.5:1 이상
- [ ] 라이트/다크모드: 모두 대비 충분
- [ ] 키보드 네비게이션: 모든 인터랙티브 요소 접근 가능
- [ ] 입력 필드: 모두 라벨 있음

### 디자인 일관성

- [ ] 버튼 색상: primary/secondary/destructive 일관
- [ ] 배지 색상: 카테고리별 일관
- [ ] 아이콘: 크기(h-4 w-4), 색상(text-muted-foreground) 일관
- [ ] 여백: space-y-_, p-_ 패턴 일관
- [ ] 타이포그래피: 제목/본문/설명 크기 일관
