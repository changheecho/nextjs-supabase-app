# 소모임 통합 관리 웹앱 MVP - Lean Canvas

## 1. Problem (문제점)

**현재 상황**: 소규모 모임(5~20명) 주최자들이 아래 도구들을 병행 사용하며 운영 부담 증가

| 도구              | 사용 목적                 | 문제점                                      |
| ----------------- | ------------------------- | ------------------------------------------- |
| 카카오톡 단톡방   | 공지 전달, 참여 의사 확인 | 메시지 유실, 응답 추적 불가, 카풀 신청 혼선 |
| 구글 스프레드시트 | 참여자 명단, 정산 내역    | 접근 권한 관리 번거로움, 모바일 편집 불편   |
| 개인 계좌 문자    | 계좌 안내, 입금 확인      | 수동 확인 반복, 실수 발생                   |

**핵심 문제**: 컨텍스트 스위칭으로 주최자 운영 부담 ↑, 참여자도 어디서 무엇을 확인할지 불명확

---

## 2. Customer Segments (고객 세그먼트)

### 주 사용자 - 모임 주최자

- **프로필**: 정기적으로 5~20명 규모 소모임 운영하는 일반인
- **현재 도구**: 카카오톡 + 스프레드시트 병행 사용 중
- **기술 친숙도**: 보통 (스마트폰 앱 설치, 온라인 뱅킹 사용 가능)
- **통증 심각도**: 높음 (반복 작업으로 30분 이상 낭비)

### 부 사용자 - 모임 참여자

- **프로필**: 주최자가 공유한 초대 링크로 진입하는 모임 구성원
- **사용 방식**: 웹 브라우저 링크 클릭 → 참여 신청 → 정보 확인
- **설치 부담**: 없음 (앱 설치 불필요)

---

## 3. Unique Value Propositions (고유 가치 제안)

### 주최자 관점

1. **한 곳에서 끝내기**
   - 공지, 참여자 관리, 카풀, 정산을 하나의 앱에서 처리
   - 카카오톡 + 스프레드시트 + 문자 왕복 제거

2. **주최자 중심 UX**
   - 초대 링크 하나로 전체 운영 자동화
   - 참여자 승인, 공지 발송, 정산 계산을 몇 번의 클릭으로 완료

3. **시간 절감**
   - 모임 운영 반복 작업 30분 이상 단축
   - 참여자 응답 자동 추적 (수동 확인 제거)

### 참여자 관점

4. **링크 하나로 전부**
   - 별도 앱 설치 없음
   - 모임 정보, 공지, 카풀, 정산을 한 페이지에서 확인

5. **신뢰할 수 있는 정보**
   - 정확한 참여 상태 (승인/대기/거절)
   - 계좌 정보와 정산 명세 공식 기록

## 4. Solution (해결책)

### MVP 기능 (Phase 1)

| 기능            | 주요 기능                                                        | 사용자 |
| --------------- | ---------------------------------------------------------------- | ------ |
| **모임 생성**   | 제목, 날짜, 장소, 최대 인원, 계좌 정보 입력 + 초대코드 자동 생성 | 주최자 |
| **초대 링크**   | 비인증 사용자도 링크 클릭 가능, 로그인 후 자동 참여 신청 처리    | 모두   |
| **참여자 관리** | 신청자 상태 조회, 승인/거절/탈퇴 처리, 모임 마감 기능            | 주최자 |
| **공지 작성**   | 제목, 내용, 핀 고정 여부 지정 가능, 참여자 전체에 공지           | 주최자 |
| **모임 홈**     | 핀 공지 요약, 기본 정보, 인원 현황, 초대 링크 복사               | 모두   |

### 추가 기능 (Phase 2)

- **카풀 제공 등록**: 드라이버가 출발지, 출발 시각, 탑승 가능 인원 등록
- **카풀 신청 관리**: 탑승자 신청, 드라이버 수락/거절 처리
- **정산 항목 등록**: 균등 분할 또는 수동 입력, 자동 금액 계산
- **입금 확인 처리**: 주최자가 각 참여자 입금 여부 체크

## 5. Key Metrics (핵심 지표)

### Phase 1 완료 후 4주 기준

| KPI                      | 목표      | 측정 방법                        |
| ------------------------ | --------- | -------------------------------- |
| 생성된 모임 수           | 10개 이상 | Supabase events 테이블 COUNT     |
| 모임당 평균 참여자 수    | 8명 이상  | event_members 승인 상태 평균     |
| 초대 링크 참여 신청 비율 | 80% 이상  | /join/[code]를 통한 신청 비율    |
| 주최자 재사용률          | 60% 이상  | 2회 이상 모임 생성한 사용자 비율 |
| 모임 운영 시간 단축      | 30분 이상 | 주최자 피드백 기반 정성 평가     |

## 6. Channels (고객 접근 채널)

### 초기 고객 확보

1. **개인 네트워크**
   - 팀 멤버의 모임 주최자 친구, 지인
   - SNS (인스타그램, 페이스북) 친구 그룹

2. **커뮤니티 활동**
   - 서울 지역 모임/동호회 커뮤니티
   - 스포츠/문화 카카오톡 오픈 채팅
   - Reddit 한국 서브레딧 (r/korea)

3. **소셜 미디어**
   - 제품 헌트(Product Hunt) 출시
   - 한국 스타트업 커뮤니티 (SparkLabs, 스타트업 페이스북 그룹)
   - 트위터/X에서 "모임 관리" 키워드 타게팅

4. **주최자 플랫폼 파트너**
   - 같이 가요(프로그래밍 모임 플랫폼) 등 연동
   - 모임 관련 웹사이트에 광고 또는 제휴

| **결제 연동 수수료** | 카카오페이/토스페이 연동 시 거래액의 3%    | 변동       |

## 9. Unfair Advantage (부당한 이득 / 차별화 요소)

### 기술적 차별화

1. **빠른 개발 속도**
   - Next.js 16+ + Supabase 조합으로 신속한 MVP 출시
   - 기존 인증 시스템 재활용 (Google OAuth 기구현)

2. **초대 링크 UX**
   - 비인증 사용자도 로그인 없이 모임 정보 미리보기 가능
   - 로그인 후 자동 참여 신청 처리 (One-Click Join)
   - 기존 카카오톡 플로우와의 자연스러운 연결

### 비즈니스 차별화

3. **주최자 중심 설계**
   - 카카오톡 + 스프레드시트 통합으로 "정말 필요한 기능"에만 집중
   - 불필요한 기능 제거로 사용성 극대화

4. **첫 사용까지의 시간**
   - 앱 설치 불필요
   - 초대 링크 클릭 → 로그인(또는 가입) → 참여 신청 (3단계, 3분 소요)

### 네트워크 효과

5. **초대 기반 바이럴 루프**
   - 주최자가 초대 링크 공유 → 참여자들이 자동으로 플랫폼에 유입
   - 참여자가 나중에 본인 모임 주최 시 플랫폼 사용 (재사용률 극대화)

## 10. Validation & Experiments (검증 및 실험)

### MVP 출시 전 검증

- [ ] 주최자 5명 + 참여자 40명으로 실제 모임 1회 운영 테스트
- [ ] 핵심 UX (초대 링크 → 로그인 → 참여 신청) 피드백 수집
- [ ] 모임 마감 후 NPS(Net Promoter Score) 조사

### 성공 기준

- NPS ≥ 50 (적극 추천 고객 비율)
- 재사용 의향 ≥ 80% ("다음 모임에서도 사용할 의사")
- 초대 링크 클릭율 ≥ 70% (비인증 사용자 → 로그인 전환)

## 11. Problem-Market Fit 검증

### 가설

> "소규모 모임 주최자는 카카오톡 + 스프레드시트 대신 단 하나의 웹앱으로 운영할 수 있다면, 30분 이상 시간을 절약할 수 있고, 참여자 응답도 더 정확해질 것이다."

### 검증 방법

1. **정성적 피드백** (주에 2~3명 인터뷰)
   - "어떤 부분이 가장 마음에 드나?"
   - "뭐가 부족했나?"
   - "친구들에게 추천할래?"

2. **정량적 지표** (MVP 사용 데이터)
   - 초대 링크 전환율 (비인증 → 가입)
   - 공지 열람 시간 (몇 초 안에 확인하는지)
   - 참여 신청 ~승인까지 평균 시간

3. **주최자 만족도**
   - 모임 운영 시간 단축 정도 (실제 측정)
   - 카카오톡 메시지 수 감소 (스크린샷 기반)

## 12. Go-to-Market Strategy (출시 전략)

### Phase 1 MVP 출시 (2026-03-22)

**타겟**: Early Adopter 100명 (주최자 10명 + 참여자 90명)

1. **Week 1~2**: 내부 테스트 + 팀 네트워크 활용
   - 팀 멤버 친구 모임 주최자 5명 모집
   - 각 모임 평균 10명 참여자

2. **Week 3~4**: 피드백 수집 + 공개 출시
   - 제품 헌트에 출시
   - SNS (트위터, 인스타) 공유
   - 한국 스타트업 커뮤니티에 포스팅

### Phase 2 (2026-04-26)

- 카풀 + 정산 기능 추가
- 유료 플랜 추가 (프리 → 프로)
- 마케팅 강화 (Google Ads, 카카오 디스플레이 광고)
